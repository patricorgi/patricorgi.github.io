iOS 及 Mac OS 科学上网方法分享其一

iOS 及 Mac OS 科学上网方法分享其一

Across the GFW, we can reach every corner in the world.

GFW 是 Great Fire Wall的缩写，这种说法实在有些讽刺意味。在此分享一个适用于 Mac OS X 及 iOS 9 系统的翻越这座 GFW （或称科学上网）的方法。

PS: 科学上网的方法有很多，这篇虽说是“其一”，但以后是否会分享“其二”，我也说不准。

然后进入正题，这个方法会涉及到 Hosts 和 Surge 两个概念。

Hosts 文件

可能大家对 Hosts 都不陌生，我就简单搬运一下 Wikipedia 的介绍

﻿Hosts 是一个用于储存计算机网络中各节点信息的计算机文件。这个文件负责将主机名称映射到相应的IP地址。hosts文件通常用于补充或替换网络中DNS的功能。和 DNS 不同的是，计算机的用户可以直接对 hosts 文件进行控制。

使用 Windows 系统的时候我就经常接触到 hosts 文件的设置，比如我通常使用 Hosts 来屏蔽优酷等视频网站的广告，或者通过对节点优化来加速上网等，windows 和 linux 系统均可通过修改 hosts 文件来配置网络节点。Mac OS 的 hosts 文件则是存放在 \etc\ 目录下。

但是，今天的主角是 Surge，这里之所以提到 Hosts 只是为了让大家更好地去理解 Surge 而已。

说到底，什么是 Surge ？

Surge 可以看做是 iOS 版的 Hosts，它提供了 iOS 和 Mac OS 上的网络节点配置功能。它完整利用了 iOS 系统最新的网络代理特性，可通过规则设定对不同 App 的网络连接进行监控和调试，因此功能比Hosts更为强大。在iOS的 App Store 上是一枚付费应用，售价为￥68，并附赠 Mac Beta版（利用Surge for iOS激活使用）。

配置 Surge 实现科学上网

然而Surge仅仅是一个配置工具，并不提供科学上网所必须的网络节点，且不幸的是，许多这样的节点服务都是要付费的。

好在有人为我们贡献了一份由txthinking/google-hosts、racaljk/hosts维护的Surge 节点列表，名曰 Hosts for Surge！

Host 文件地址：Hosts for surge

这是一份按照 Surge 的配置文件规则生成的 Hosts 列表，维护人员大约每3天更新一次，不用担心某一天突然就失效了。支持网站主要有 Wikipedia、Google、Facebook、Twitter、Instagram等等。

它的配置十分简单。

Surge for iOS 配置

首先打开 Surge for iOS，可以看到如下界面

点击Download Configuration from URL，输入Hosts for surge链接的地址，然后点击OK即可。

之后配置文件列表中会出现hosts条目，选中它，再点击右上角的done，接着点击Start，手机就自动开启VPN了。

Surge for Mac 配置

  1. 先下载Surge的Mac版，点击这里
  2. 将其安装到应用程序文件夹中
  3. 利用终端在创建~/.surge.conf文件，并复制Hosts for surge页面中的内容到.surge.conf文件中
  4. 启动Surge.app，并保证Mac以及安装有Surge for iOS的移动设备均开启了Wi-Fi，然后在移动端Surge里进入More → Surge Mac Beta → Tap to activate
  5. 之后在Surge for Mac上打开Set as System Proxy

至此我们成功利用 Surge 实现了科学上网
